    • Subagent Tool Lazy Loading
      • Full support for subagent tool lazy loading, each agent’s loaded tool state is isolated
      • ToolLoader / ToolUnLoader isolates loaded state by agentId, subagent won’t pollute main agent’s tool list
      • Fork mode subagent passes complete original tool list (_tools), ensuring lazy loading can access all available tools
      • Tool.prompt adds optional context parameter, ToolLoader can sense current agent’s tool scope
      • In lazy loading mode, ToolLoader / ToolUnLoader are always available (fixTools dynamically supplements based on enableLazyTool)
      • Unloaded tool error hints also work for subagents
